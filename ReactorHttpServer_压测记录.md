# ReactorHttpServer 压测记录

## 1. 测试目的

验证 ReactorHttpServer 在 Linux 环境下处理静态 HTTP 请求的并发能力、稳定性和文件发送性能，并对比普通文件发送路径与零拷贝 `sendfile` 路径的性能差异。

## 2. 测试环境

| 项目 | 配置 |
| --- | --- |
| 测试机器 | 远程 Linux 虚拟机 |
| 服务端地址 | `127.0.0.1:10000` |
| 测试方式 | 本机回环压测 |
| 编译方式 | `g++ -std=c++11 -O2 *.cpp -o reactor_http -pthread` |
| 服务端线程数 | 4 个工作线程 |
| 文件描述符限制 | `ulimit -n = 1024` |
| 压测客户端 | Python 并发 HTTP 客户端 |

说明：本次测试为本机回环环境，主要用于验证项目并发处理能力和不同发送路径的相对性能差异，不等同于跨机器网络环境下的生产压测结果。

## 3. 测试对象

### 小文件请求

测试路径：

```text
/index.html
```

文件内容：

```text

```

## 4. 阶梯并发压测结果

小文件 `/index.html` 阶梯压测结果如下：

| 并发数 | 请求数 | QPS | 平均延迟 | P95 延迟 | P99 延迟 | 失败请求 |
| ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| 200 | 40,000 | 15,585.02 | 0.83 ms | 1.88 ms | 2.65 ms | 0 |
| 400 | 80,000 | 15,345.39 | 1.42 ms | 3.53 ms | 5.05 ms | 0 |
| 600 | 120,000 | 15,897.69 | 1.86 ms | 4.75 ms | 7.56 ms | 0 |
| 800 | 160,000 | 16,365.13 | 2.28 ms | 5.91 ms | 8.31 ms | 0 |
| 1,000 | 150,000 | 16,304.29 | 2.17 ms | 5.86 ms | 8.40 ms | 0 |
| 1,500 | 225,000 | 15,853.82 | 3.30 ms | 10.46 ms | 19.95 ms | 0 |
| 2,000 | 300,000 | 16,237.11 | 4.08 ms | 15.01 ms | 25.34 ms | 0 |
| 3,000 | 300,000 | 16,173.09 | 4.09 ms | 15.30 ms | 25.66 ms | 0 |
| 5,000 | 500,000 | 15,253.90 | 7.30 ms | 24.70 ms | 32.50 ms | 0 |
| 8,000 | 800,000 | 14,323.81 | 13.26 ms | 30.62 ms | 41.58 ms | 0 |
| 10,000 | 500,000 | 15,415.12 | 7.12 ms | 24.39 ms | 32.02 ms | 0 |

### 结果观察

- 在 800-3,000 并发区间，QPS 基本稳定在 1.6 万左右。
- 最高测试到 10,000 并发、500,000 次请求，失败请求数为 0，服务端进程保持正常运行。
- 随着并发数继续上升，平均延迟和 P95/P99 延迟明显增加，说明压力逐渐转向客户端调度、系统资源和回环压测开销。
- 本次测试没有观测到服务端崩溃或请求失败。

## 5. 零拷贝发送对照测试

测试对象：`/1mb.txt`，文件大小 1 MB。

测试条件：100 并发，1,000 次请求。

| 发送方式 | QPS | 吞吐量 | 平均延迟 | P95 延迟 | P99 延迟 | 失败请求 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| 零拷贝 `sendfile` | 3,815.53 | 3,815.84 MB/s | 0.93 ms | 1.52 ms | 2.06 ms | 0 |
| `read -> Buffer -> send` | 2,656.75 | 2,656.96 MB/s | 1.57 ms | 5.46 ms | 9.38 ms | 0 |

### 对比结果

以 `read -> Buffer -> send` 作为基准，零拷贝 `sendfile` 路径表现为：

| 指标 | 优化效果 |
| --- | ---: |
| 吞吐量 | 提升约 43.6% |
| 平均延迟 | 降低约 40.8% |
| P95 延迟 | 降低约 72.2% |

## 6. 可用于简历的表述

推荐简历表述：

```text
压测验证：在 Linux 本机回环环境下进行阶梯压测，最高测试 10000 并发、50 万次请求，失败请求数为 0；2000 并发下 QPS 约 1.62 万、平均延迟约 4.08 ms。针对 1MB 静态文件，零拷贝 sendfile 相比 read -> Buffer -> send 吞吐量提升约 43.6%。
```

更保守版本：

```text

```

## 7. 注意事项

- 本次压测客户端为 Python 并发 HTTP 客户端，不是 `wrk` / `ab` 等专业压测工具。
- 测试环境为本机回环，结果主要反映服务端在当前虚拟机环境下的相对性能和稳定性。
- 继续提高并发时，结果可能更多受到 Python 客户端、虚拟机调度、文件描述符限制等因素影响。
- 若后续使用 `wrk` 或跨机器压测，建议重新记录正式压测结果。
